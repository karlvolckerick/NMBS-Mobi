The Challenge

NMBS Contact Centres Are Under Pressure

📞
High Repetitive Volume
Millions of inbound calls per year for journey planning, timetables, and platform info. The majority of these are routine queries that don't require a human specialist — yet consume the same agent time.
⏱️
Disruption Peaks
A single major delay or cancellation creates call spikes that overwhelm the contact centre. Customers face long hold times at exactly the moment they need help most — damaging trust.
🌍
Multilingual Expectations
Belgian rail passengers speak Dutch, French, German, and English. Providing consistent quality across all four languages requires significant staffing — and night/weekend coverage is expensive.
🏗️
Legacy IVR Frustration
IVR menus force callers into rigid trees that often don't match their actual need. Misrouted calls waste agent time and drive high abandonment rates. Customers expect to speak naturally.
"Customers calling about a delayed train don't want to press 2 for disruptions. They want an answer in 10 seconds."

Solution Overview

A Production-Ready Starting Point — Already Live

The NMBS AI Contact Centre is a fully deployed, end-to-end Azure application — built on Microsoft's open-source AI Contact Centre Solution Accelerator. It answers real inbound calls on a Belgian phone number today.

Four specialist AI agents handle journey planning, ticketing, live disruptions, and digital support. Each agent connects to real-time NMBS train data and can send callers a follow-up email with their timetable, ticket confirmation, or disruption summary.

  • Live on a real Belgian phone number via Azure Communication Services
  • Connected to the iRail open API for real-time departures and connections
  • Silent agent handoffs — callers hear no "please hold" or menus
  • 600+ Azure TTS voices — multilingual ready
  • Deployed on Azure Container Apps — scales to zero when idle
🎯 What it handles today
Journey planning, live departures, connection changes, delay compensation, ticket queries, NMBS app support — all without a human agent.
⚡ What it replaces
IVR trees, hold queues, and repeated agent re-routing for common enquiry types. The AI handles the first line of contact.
🔧 What can be added
CRM integration, booking system tools, Dutch/French language configuration, warm transfer to human agents, recorded calls for QA.

Multi-Agent Design

Silent, Seamless Handoffs

Each agent is a specialist with its own voice, instructions, and tool set. Handoffs are completely silent — the caller hears a natural conversation without hold music, IVR re-entry, or re-stating their problem. The new agent receives the full call history.

# Live NMBS call example
Receptionist: "Welcome to NMBS, how can I help?"
Caller: "My Bruges to Brussels train is cancelled"
→ Silent handoff to Disruptions Agent
Disruptions: "I can see IC 513 is cancelled. The next
    alternative departs at 14:22 via Ghent…"
Caller: "Can I get compensation? And I need a refund"
→ Silent handoff to Ticketing Agent
Ticketing: "Happy to help. Can I have your account number?"
  • Full conversation history passed to each new agent
  • Agents triggered automatically by AI context awareness
  • Each agent has its own voice and personality
  • Handoff rules configured entirely in YAML — no code changes
Receptionist Agent
Greets and triages every caller. Voice: friendly, multilingual neutral. Tools: live departures, journey planning, routing to specialists.
Disruptions Agent
Delay and cancellation expertise. Voice: reassuring (en-GB-RyanNeural). Tools: iRail disruptions API, alternative route finder, email summary.

Cost Model

Indicative Pricing — Azure VoiceLive / GPT Realtime

Estimated monthly cost at NMBS scale
~$85,000
500,000 calls/month · 5-minute average call length

The dominant cost driver is Azure VoiceLive / GPT-4o Realtime audio tokens. Audio is priced at $100/M tokens input and $200/M tokens output — at 1,500 tokens per audio minute, a 5-minute call generates approximately $0.90 in model cost alone. At 500k calls that's ~$450k model cost before discounts — enterprise agreements and Azure Committed Use discounts significantly reduce this figure.

⚠️ Key Cost Variables
📞 Call length 📊 Calls per month 🔧 Number of tool calls 🤖 Model used (GPT-4o vs 4.1) 🤝 Enterprise discount 🌍 Agent language mix 📝 System prompt length ✉️ Email sends
💡 Cost Optimisation Levers
Shorter agent responses reduce output audio tokens (the single biggest line item at $200/M). Batching tool calls, caching timetable data, and auto-deflecting fully-answerable enquiries without voice reduces per-call cost.
📉 Infrastructure Cost (flat)
Azure Container Apps scales to zero when idle — ~$5–20/month at low volume, ~$50–150/month at scale. ACS phone number rental: ~$2/month + $0.004/min inbound PSTN. Not the cost to optimise.
